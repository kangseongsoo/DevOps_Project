[DevOps 프로젝트 진행 순서]

1단계: Redis 서비스 설정 및 도커화
- Redis 설정 파일(redis.conf) 작성
- Redis 도커 이미지 빌드 및 실행
- Redis 연결 테스트
- Redis 데이터 지속성 설정

2단계: 챗봇 API 개발
- FastAPI 기반 챗봇 API 개발
- Redis 연동 구현
- API 엔드포인트 구현
  * 채팅 메시지 처리
  * 대화 기록 저장/조회
  * 사용자 인증
- API 테스트 작성 및 실행

3단계: 웹 프론트엔드 개발
- React 기반 웹 인터페이스 개발
- 채팅 UI 구현
- API 연동
- 사용자 인증 UI
- 반응형 디자인 적용

4단계: 로컬 환경 통합 테스트
- Redis, API, 프론트엔드 로컬 연동
- 전체 기능 테스트
- 성능 테스트
- 버그 수정 및 최적화

5단계: 도커 컨테이너화 및 연동
- 각 서비스 도커파일 작성
  * Redis 서비스
  * 챗봇 API 서비스
  * 웹 프론트엔드 서비스
- docker-compose.yml 작성
- 컨테이너 간 네트워크 설정
- 환경 변수 설정

6단계: Nginx 리버스 프록시 설정
- Nginx 설정 파일 작성
- SSL/TLS 설정
- 로드 밸런싱 설정
- 캐싱 설정

7단계: 최종 테스트 및 배포
- 전체 시스템 통합 테스트
- 보안 검사
- 성능 모니터링
- 배포 스크립트 작성
- 문서화

각 단계별 체크리스트:
□ Redis 서비스 설정 완료
□ 챗봇 API 개발 완료
□ 웹 프론트엔드 개발 완료
□ 로컬 테스트 완료
□ 도커 컨테이너화 완료
□ Nginx 설정 완료
□ 최종 테스트 완료
□ 배포 준비 완료

참고사항:
- 각 단계별로 커밋 메시지를 명확하게 작성
- 테스트 코드는 각 기능 개발과 동시에 작성
- 문서화는 진행하면서 지속적으로 업데이트
- 보안 관련 설정은 처음부터 고려
